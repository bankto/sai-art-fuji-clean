# 仕様書を元に実装する(主担当: Codex)

`docs/workflows/implement.md` を読み、その手順に従って実装する。
対象仕様書の指定が無ければ docs/specs/ から確認する。

- 通常は Cursor(オーケストレーター)から `codex exec` でマイルストーン単位で委譲されて実行する。指示に含まれる「読むファイル・出力先・前提」に従う
- 非対話実行ではユーザーに質問できない。仕様に無い機能は足さず、必要だと思った点は「未決事項」として出力する。単体実行(対話)でも着手後は質問で止まらず完走する
- レビュー指摘(`docs/reviews/`)の修正もあなたの担当。status.md の更新は、委譲時はオーケストレーターが行い、単体実行(ユーザーからの直接依頼)時は自分で行う
