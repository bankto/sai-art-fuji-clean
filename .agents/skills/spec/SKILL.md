# 仕様書を作成する(主担当: Codex)

`docs/workflows/spec.md` を読み、その手順に従って仕様書を作成する。
対象アイデアの指定が無ければ docs/ideas/ のステータス「採用」から確認する。

- 通常は Cursor(オーケストレーター)から `codex exec` で委譲されて実行する。指示に含まれる「読むファイル・出力先・前提」に従う
- 非対話実行ではユーザーに質問できない。判断できない点は仕様書の「未決事項」に必ず書く
- レビュー指摘(`docs/reviews/`)の反映もあなたの担当
