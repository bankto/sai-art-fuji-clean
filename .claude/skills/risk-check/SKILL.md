# risk-check — 古さ・根拠・リスクを点検する

意思決定の前に、地雷を踏まないための軽量チェック。結果は `03_fact_check.md` に追記する。

## チェック項目
1. **鮮度**: 価格・営業状況・開催日・仕様・法律・SEO・AI機能など、変わりやすい情報が古くないか。最終確認日はいつか。
2. **根拠**: 出典なしの主張、個人ブログのみの主張はないか。
3. **反対意見**: 都合の良い情報だけを集めていないか。反証・失敗事例はあるか。
4. **実務リスク**: これを信じて動くと事故る点（閉業済み、予約不可、地域規制、著作権など）。
5. **飛躍**: 少数事例から一般化していないか。

## 出力フォーマット（03_fact_check.md へ追記）
```
## リスクチェック
- 鮮度リスク: （対象 / 最終確認日 / 要再確認か）
- 根拠不足: （対象 / 不足内容）
- 反対意見の欠落: （見落としている反証）
- 実務リスク: （事故りうる点）
- 飛躍: （一般化しすぎている箇所）

## 判定
Go / 条件付きGo / 要追加調査
```

## 注意
- 「疑わしきは要検証」で残す。無理に安全と結論づけない。
- 重い検証・別モデルでの精査が必要なら fact-checker を別コンテキストで起動する。
