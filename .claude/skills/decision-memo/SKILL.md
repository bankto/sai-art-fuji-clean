# decision-memo — 最終判断メモを作る

リサーチを「その場限り」で終わらせず、実装・記事・企画にそのまま流用できる資産にする。`04_decision.md` に保存する。

## 前提
- `01`〜`03` が揃っていること。検証が未了なら先に risk-check / fact-checker を通す。
- fact-checker が却下・要注意とした主張を、結論の根拠に使わない。

## 出力フォーマット（04_decision.md）
```
# Decision Memo

## 結論
採用 / 不採用 / 保留（1行）

## 理由
（3〜5点）

## 根拠（重要ソース）
- URL / 一次情報か

## 採用する事例・方向性
- （活かす参考事例・ニッチスポット）

## リスク・不確実な点
-

## 反対意見・別案
- （意見が割れた論点は隠さず残す）

## 次にやること
→ 05_next_actions.md に箇条書き
```

## 注意
- 割れた論点を多数決で握りつぶさない。「割れている」と明示する。
- 結論は1行で断言する（採用/不採用/保留）。判断を保留する場合も「何が揃えば決められるか」を書く。
