# source-matrix — 情報源を信頼度で分類する

公式情報と個人ブログを同列に扱わないための手順。`01_raw_sources.md` の各ソースを分類する。

## 信頼度の区分
| 区分 | 例 | 扱い |
| --- | --- | --- |
| 公式（一次） | 公式サイト、公式SNS、プレス、公的統計 | そのまま根拠に使える |
| 準公式 | 業界団体、自治体観光協会、提携メディア | ほぼ信頼、日付だけ確認 |
| メディア | 新聞・雑誌・専門メディア | 発行日と一次ソースを確認 |
| 個人ブログ | 個人の体験談・レビュー | 参考。事実は公式で裏取り |
| SNS | 口コミ、投稿 | 傾向把握のみ。事実確定に使わない |
| 推測 | AI/自分の推定 | 根拠にしない。要検証フラグ |

## 手順
1. `01_raw_sources.md` を読む。
2. 各ソースを上表のいずれかに分類し、下記の表を `02_analysis.md`（または `01_raw_sources.md` 末尾）に追記する。
3. 「公式で裏が取れていない重要な主張」を洗い出し、fact-checker / official-source-researcher に回す候補として明記する。

## 出力フォーマット
```
## 情報源マトリクス
| 主張・事例 | 情報源 | 区分 | 発行/更新日 | 一次裏取り | 古い可能性 |
| --- | --- | --- | --- | --- | --- |
```

## 注意
- 1つの主張が個人ブログのみに依存している場合、必ず「要公式確認」とマークする。
