# research-brief — 調査ブリーフを作る

収集を始める**前**に、何を調べ・何を調べないかを固める。ここが曖昧だと収集が際限なく広がる。

## 手順
1. トピック名を決め、フォルダ `docs/research/YYYY-MM-DD_テーマ名/` を用意する(雛形: `docs/research/_templates/full/`)。
2. 下記フォーマットで `00_brief.md` を **ファイルとして作成する**（会話で終わらせない）。
3. 「調べないこと」を必ず1つ以上書く。

## 00_brief.md フォーマット
```
# Research Brief

## 調査テーマ
（何を調べるか）

## 背景
（なぜ調べるか）

## 最終的に決めたいこと
（例: 採用する参考事例を決めたい / ニッチ候補を10件出したい / 方向性を選びたい）

## 調査範囲
- 国内 / 海外の別:
- カテゴリ（宿 / 施設 / イベント / 表現 など）:
- ニッチの定義（該当する場合）:

## 調査しない範囲
（今回は見ないもの）

## 判断基準
（何をもって良し悪しを決めるか。独自性 / 再現性 / 費用 / 信頼性 など）

## 成果物
（最終的にどの md にまとめるか）
```

## 注意
- 海外を対象に含むなら「英語・現地語ソースまで当たる」と明記する。
- ニッチ調査なら「除外する定番・大手」を具体的に書く。
