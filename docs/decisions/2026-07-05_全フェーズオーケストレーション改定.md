# 決定: 全フェーズ共通の AI オーケストレーション体制へ改定

- 決定日: 2026-07-05
- 決定者: ユーザー(hitomi)
- 実施: Claude Code(運用ルール改定作業として、例外的に対象ファイルを直接編集)

## 変更理由

1. ユーザーの拠点を Cursor Agents Window に移し、iOS アプリ(Remote Control / Cloud Agent)からの遠隔指示・監視を可能にするため。遠隔操作では「1つの窓口に依頼すれば、あとはAI側で最適な担当に振り分けて完遂する」体制が必須になる
2. 従来は Phase 4〜5 のみ役割固定・Phase 1〜3 は「任意のAI」だったため、フェーズによって指示の出し方が変わり、遠隔運用の妨げになっていた
3. Phase 1(リサーチ)の実績で、実質的に「Claude Code=収集・検証 / Codex=整形・実装 / Cursor=段取り」の分担が既に機能していた。実態に合わせてルールを明文化する

## Before / After

### Before(〜2026-07-04)

| フェーズ | 体制 |
|---|---|
| Phase 1〜3 | 役割固定なし。任意のAIが実施 |
| Phase 4〜5 | Cursor=オーケストレーター / Codex=作成 / Claude Code=レビュー |

### After(2026-07-05〜)

**フェーズに関係なく常に同じ体制:**

| ツール | 常時の役割 |
|---|---|
| **Cursor(Agents Window)** | **オーケストレーター**。タスク分解・CLI委譲・進行管理・検証・ユーザー報告・`docs/status.md` 更新。自分で大きな作成/実装はしない |
| **Codex** | **作成担当**。ドキュメント執筆、コード/サイト実装、CSV生成、レビュー指摘の反映、データ整形 |
| **Claude Code** | **調査・検証・レビュー担当**。リサーチ収集(専門調査員)、ファクトチェック(収集とは別セッション)、仕様/実装レビュー。対象ファイルの直接修正は原則せず、レビューは `docs/reviews/`、調査は `docs/research/` に書く |

- フェーズ番号(1〜5)と、成果物の置き場所・雛形・`docs/status.md` 更新ルールは従来どおり維持
- ユーザーが特定のAIに直接依頼した場合(単体実行)は従来どおり可能。ただし各AIは自分の役割を踏まえ、役割外の作業は役割分担を伝えたうえでユーザーの指示に従う
- オーケストレーションの正本は `docs/workflows/orchestrate.md`(全フェーズ版に拡張)。各入口(コマンド/スキル)はこれを参照する薄いラッパーのまま

### 付随する整理

- デザインモック・プロトタイプ等の「作成系」も原則 Codex に寄せる(2026-07-05 のこの改定が、それ以前の個別のやりとりより優先)。ユーザーが特定AIを指名した場合はそれに従う
- `claude -p` は呼び出しごとに新しいセッションになるため、「収集と検証(fact-check)を別セッションに分ける」ルールは**別々の CLI 呼び出しにすることで充足**する。1回の呼び出しに収集と検証を混ぜることは禁止のまま

## ユーザーが Cursor Agents で使う定番起動プロンプト(コピペ用)

### 基本(全フェーズ共通・これだけ覚えればよい)

```
AGENTS.md と docs/status.md と docs/workflows/orchestrate.md を読んで、オーケストレーターとして動いて。作成は Codex、調査・検証・レビューは Claude Code に CLI 委譲。軽い確認と status 更新は Cursor 自身。ユーザー承認が必要な判断は止まって聞いて。
```

### 続きから再開(セッションをまたぐとき)

```
docs/status.md を読んで現状を把握して、オーケストレーターとして続きを進めて。委譲ルールは docs/workflows/orchestrate.md に従って。
```

### 具体的な依頼を添える場合の例

```
(基本プロンプトに続けて)
今日やりたいこと: ○○のリサーチを本格モードで始めたい。ブリーフの壁打ちから。
```

```
(基本プロンプトに続けて)
research-site のデザインを修正したい。要件: ○○。実装は Codex に委譲して、結果を検証して報告して。
```

スラッシュコマンド派は `/orchestrate`(全フェーズ共通)または各フェーズの `/research` `/research-site` `/ideate` `/spec` `/implement` でも同じ動きになる。

## 移行時の注意

1. **CLI 認証**: 委譲は「エージェントが動いているマシン」で `codex` / `claude` CLI が実行できることが前提。デスクトップで `codex --version` / `claude --version` とログイン状態を事前確認する
2. **iOS からの利用形態で挙動が変わる**:
   - **Remote Control(デスクトップのエージェントを iOS から遠隔操作)**: デスクトップの CLI をそのまま使えるため、委譲を含む全機能が動く。**推奨形態**
   - **Cloud Agent(クラウド実行)**: 実行環境に `codex` / `claude` CLI が無い・未認証の可能性が高い。その場合オーケストレーターは委譲せず、軽作業のみ行い「デスクトップ(Remote Control)で再開が必要」と報告して止まるルール(orchestrate.md に明記)
3. **git が未初期化**(2026-07-04 発見・未対応): `.git/` が空でバージョン管理が効いていない。今回のような大規模改定の差分管理・ロールバックのためにも `git init` + 初回コミットを強く推奨
4. **旧体制の記述**: 過去のリサーチ成果物や status.md の過去ログに残る「任意のAI」等の記述は履歴のため修正しない
5. **タイムアウト**: `codex exec` / `claude -p` は数分かかることがある。Agents Window 側のタイムアウト設定に注意し、長大なタスクはマイルストーン単位に分割して委譲する
