# 認識モデル用：出張撮影チェックリスト

- 日付: 2026-07-13
- 用途: 会場で実物ゴミを撮影し、認識モデルを作るための現場メモ
- 対象アプリ: `apps/camera-ar-gomi-demo/`
- 用語: 本メモでは「クラス」と呼ばず、すべて **種類** と書く

## 用語

| 言葉 | 意味 |
|---|---|
| 種類 | アプリに見分けさせたいゴミ（または背景）の区分。写真を入れる箱の名前 |
| 枚数 | その種類ごとに撮る写真の数 |
| 背景 | ゴミが写っていない状態用の種類（机・台だけ） |

例: 「アルミ片」「廃プラスチック」「紙くず」「背景」の4種類。

## 1. いちばん大事なこと

1. **なんでもゴミではなく、会場で撮れる少数の種類に絞る**
2. **見た目が大きく違うもの同士**を選ぶ（似た色・似た形は誤認識しやすい）
3. **背景という種類を必ず1つ入れる**
4. **本番と同じ条件で撮る**（照明・距離・背景・スマホ）
5. 現場は写真の確保が最優先。学習は帰宅後でよい

## 2. 種類の決め方（何があるか分からないとき）

1. 最初に、撮影できそうなゴミをざっと見る
2. 見た目がはっきり違うものから **ゴミ3種類** を選ぶ
3. 余裕があればゴミを4〜5種類まで増やす（最初は増やしすぎない）
4. 必ず **背景** を1種類追加する

### 推奨スタート（合計4種類）

| 種類の例 | 現場での拾い方 |
|---|---|
| 金属っぽいもの | アルミ端材・缶片など。光沢・硬い質感 |
| プラスチックっぽいもの | ボトル片・容器など |
| 紙っぽいもの | 紙くず・段ボール片など |
| 背景 | 対象なし。机・床・台だけ |

現場では仮の名前（A金属 / Bプラ / C紙 / D背景）でもよい。あとで日本語名に揃えられる。

### 種類数の目安

| 段階 | 内容 |
|---|---|
| 最低限 | ゴミ2種類 + 背景 = 3種類 |
| 推奨 | ゴミ3種類 + 背景 = **4種類** |
| 初回の上限 | ゴミ5種類 + 背景 = 6種類 |

### 避けた方がよい選び方

- 同じ色のプラと紙（白い同士など）
- 小さくて特徴の少ない破片だけを複数種類にする
- 「混合ゴミ」を1種類にする（中身が曖昧になる）

## 3. 枚数の目安

| 種類 | 最低 | 推奨 |
|---|---:|---:|
| 各ゴミの種類 | 30枚 | **50枚** |
| 背景 | 30枚 | **50枚** |

ゴミ3種類 + 背景なら、推奨で **だいたい200枚**（50×4）。

### 枚数より大事なこと

同じ角度の連写より、次を混ぜる:

- 距離（近い / 普通 / 少し遠い）
- 角度（正面 / 斜め）
- 置き方・裏表
- 同じ種類でも別の破片（個体差）

## 4. 撮影のコツ

### やる

- デモ本番に近い机・背景・照明で撮る
- スマホの背面カメラで撮る
- 対象を画面の中央寄りに収める
- 種類ごとにフォルダ分けして保存

### やらない

- 極端な接写だけ、または遠景だけ
- 真っ黒・真っ白になる条件だけ
- 本番と全然違う部屋だけで揃える

### 背景の撮り方

- 対象をどかす
- 空の机・台を、距離と角度を変えて撮る
- ゴミが端に写っている写真は背景に入れない

## 5. 帰宅後（概要）

1. [Teachable Machine（Image）](https://teachablemachine.withgoogle.com/train/image) を開く
2. 種類ごとに写真を入れる
3. 学習（Train）する
4. Tensorflow.js 形式で書き出す（Export / Download）
5. 解凍して `apps/camera-ar-gomi-demo/public/models/` に置く  
   （`model.json` / `metadata.json` / `weights.bin` など）
6. アプリで実機確認

配置後の確認・反映は AI 側で支援できる。

## 6. 現場チェックリスト

- [ ] ゴミの種類を決めた（推奨: 3種類）
- [ ] 背景用に空の机・台を確保した
- [ ] 種類名をメモした（仮名で可）
- [ ] 各ゴミの種類を50枚前後撮った（最低30）
- [ ] 背景を50枚前後撮った（最低30）
- [ ] 距離・角度・別の破片でばらつきを入れた
- [ ] 種類ごとにフォルダ分けして保存した
- [ ] 本番に近い照明・背景で撮れた

## 7. よくある失敗

| 失敗 | 起きること | 対策 |
|---|---|---|
| 1個の破片しか撮らない | その破片しか当たらない | 同種を複数用意 |
| 背景がない | 何もないのにゴミと判定 | 背景を必ず撮る |
| 似た見た目の2種類 | 入れ替わる | 見た目差が大きい組を選ぶ |
| 枚数は多いが同じ構図 | 少し動くと外れる | 距離・角度を変える |

## 8. 帰宅後に共有してほしいもの

- 決めた種類名の一覧
- 各種類の枚数
- Teachable Machine の書き出し一式、または `public/models/` に置いた状態
- 現場メモ（暗かった、反射が強かった等）
