# history/

`docs/status.md` から退避した決定事項・AI実行ログの月別アーカイブ。status を「再開用の短いダッシュボード」に保つための置き場で、再開時に読む必要はない(過去の経緯を調べるときだけ読む)。

- ファイル名: `YYYY-MM.md`(移す行の日付の月。月をまたぐ場合は行ごとに振り分ける)
- ファイル内の見出しは `## 決定事項ログ` / `## 直近のAI実行ログ` の2つ。status と同じ表形式のまま末尾に追記する(古い行が上)
- **いつ・誰が**: status の履歴系2節がデータ行20行を超える更新をする時に、同じ更新の中でオーケストレーターが古い行から移す(新セッションの開始時・フェーズの区切りにも点検する。正本: `docs/workflows/orchestrate.md`「docs/status.md の運用」)
- **移動であってコピーではない**: 移した行は `status.md` から削除する(二重管理しない)
- アーカイブ後も成果物への相対リンク・タスクID・実行指定/確認値を保持する
- 履歴は追記専用とし、過去の意思決定を無断で書き換えない
