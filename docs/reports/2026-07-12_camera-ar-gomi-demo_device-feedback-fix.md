## 実装した内容

- デモ用の種類を変更した時点で認識を再実行し、認識バッジへ即時反映するようにした。
- デモ信頼度をラベル別の固定値（アルミ88%、廃プラスチック84%、紙79%、未認識18%）へ分け、画面では「デモ固定値」と明記した。値は実際の認識精度ではない。
- 認識バッジへ「デモ・手動選択」または「モデル認識」のモード表示を追加した。
- 「デモラベル」を「デモ用の種類（手動選択）」へ変更し、モデル未配置時の操作説明を追加した。モデル利用時は手動選択欄を隠す。
- カメラ画面を 100dvh と safe-area-inset 基準のスクロールなしレイアウトへ変更した。
- 認識更新・生成ボタンをカメラ映像上の下部オーバーレイへ移し、z-index、52px以上のタップ高さ、touch-actionを指定した。認識バッジは上部へ移し、操作帯との重なりを避けた。
- 作品画面もカメラ画面と同じ基本フォントを使い、スマートフォンではCanvasと操作パネルが画面内に収まりやすい可変レイアウトへ変更した。
- NFC、TAKEAWAY（URL・QR）、AR検証ボタンを `hidden` と `data-feature="deferred"` で一時非表示にした。DOMと処理コードは残している。
- ARのハッシュURLを直接開いた場合も、一時非表示中は作品の通常再生を表示するようにした。
- PWA利用時に古い画面が残らないよう、Service Workerのキャッシュ名を v5 へ更新した。
- 上司向けの将来機能説明資料を作成した。

## 検証結果

- lint: `npm run lint` 成功（TypeScript型チェック、2026-07-12）
- ビルド: `npm run build` 成功（Vite 7.3.6、2,287 modules transformed、2026-07-12）
- ビルド補足: lintとの連続実行時にWindows環境の子プロセス起動が `spawn EPERM` で1回失敗した。コード変更は行わず `npm run build` を単独再実行し、成功を確認した。
- テスト: 未追加。既存プロジェクトにtestスクリプト・ブラウザE2E環境がなく、新ライブラリ追加禁止のため、今回は型チェック・本番ビルド・DOM/CSS差分確認で検証した。デモ認識の変更は実機操作確認を次工程に残す。
- 依存監査・秘密情報・セキュリティ自己点検: 依存・lockfile変更なし。新規秘密情報なし。カメラ映像を外部送信する処理は追加していない。HTMLへユーザー入力を挿入する変更なし。
- 動作確認: ビルド生成まで確認済み。iPhone / Android実機でのタップ、safe area、画面回転、カメラ許可は未確認。
- 警告: Viteから500kB超のchunk警告が出る。既存のTF.js等を含む構成によるもので、ビルド自体は成功。今回は新規依存追加・構成変更をしていない。

## 自律判断ログ

- 一時非表示機能はHTMLの `hidden` / `data-feature="deferred"` と、TypeScriptの `deferredFeaturesEnabled = false` を併用した。画面導線だけでなくARハッシュ直開きも通常再生へ戻し、現在の説明範囲と表示を一致させるため。
- デモ信頼度は、ユーザーに違いが分かる範囲でラベル別固定値にした。実測値と誤認されないよう、必ず「デモ固定値」と併記した。
- カメラ・作品画面だけbodyスクロールを止め、開始画面や将来再表示するAR画面の既存挙動への影響を限定した。
- PWAキャッシュ名を更新した。既存READMEが画面変更時のバージョン更新を運用ルールとしているため。
