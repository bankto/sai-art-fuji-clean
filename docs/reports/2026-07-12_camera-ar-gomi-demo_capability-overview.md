# カメラ認識 × 生成アートデモで将来的にできること

- 日付: 2026-07-12
- 対象: 上司・プロジェクト関係者向け
- 関連仕様: [カメラ認識 × AR ゴミデモ仕様書](../specs/2026-07-10_camera-ar-gomi-demo.md)
- 技術検討: [技術実現性調査](../research/2026-07-10_camera-ar-gomi-demo-feasibility.md)

## 1. 概要

このデモは、廃棄物などの対象をスマートフォンのカメラで認識し、その結果をもとに音と映像の作品を端末内で生成する体験の試作です。作品は「Seed（作品を再現する短い識別情報）」で管理できるため、将来は NFC、QRコード、URL、画像ターゲットAR、レーザー彫刻などを組み合わせ、「現場で生まれた作品を持ち帰り、後から同じ作品を呼び出す」体験へ拡張できます。

## 2. 現在デモで見せていること

現在の画面では、次の流れに絞って体験できます。

1. スマートフォンでカメラを起動する
2. 対象の種類を認識する
3. 認識結果から Seed を作る
4. Seed をもとに、音と動くビジュアルを端末内で生成する
5. 再生・停止・音量調整・再生成・PNG保存を行う

認識モデルが未配置の環境では「デモ用の種類」を手動で選び、認識後の体験を確認します。実物を自動判別するには、対象物を撮影して作成した分類モデルの追加と実機調整が必要です。

今回、NFC、URL・QRによる持ち帰り、AR、レーザー彫刻向けの導線は画面から一時非表示にしています。ただしコードは削除しておらず、現在の体験を分かりやすくするための整理です。

## 3. 拡張できること

| 拡張 | できること | 現在地・追加検証 |
|---|---|---|
| NFC | タグに作品URLを記録し、スマートフォンをかざして同じ作品を開く | ソフトウェアの土台は実装済み。Androidでの書き込み、iPhoneを含む読み取りの実機確認が必要 |
| QR・URL持ち帰り | QRを読む、またはURLを開くことで、後日同じ作品を再生する | Seed入りURLとQR生成の土台は実装済み。別端末での再現確認が必要 |
| 画像ターゲットAR | 印刷した図案をカメラで認識し、その上に作品映像を重ねる | AR用コードは実装済み。ターゲット生成、印刷、端末別追跡テストが必要 |
| レーザー彫刻QR | 廃材や展示物に彫刻したQRから作品を呼び出す | 構想・技術候補。素材、反射、線幅、コントラストの試作が必要 |
| レーザー彫刻AR図案 | 彫刻図案を画像ターゲットとして認識し、作品を重ねる | 推測: 成立すれば作品性と機能を両立できる。素材別の追跡試験が必要 |
| 対象物の種類追加 | アルミ、廃プラスチック、紙などで異なる音・映像を出す | 分類モデルと撮影データの準備が必要。会場の照明・背景に合わせた調整が重要 |
| 作品カード・記念画像 | 生成作品をPNGとして保存し、展示記録や持ち帰り素材にする | PNG保存は現在のデモで利用可能 |
| オフライン寄りの運用 | 会場通信が弱い場合も、事前読込済み端末で体験を続ける | PWAの土台は実装済み。実運用前にキャッシュ範囲と端末挙動の確認が必要 |

## 4. 体験パターン例

以下は将来の体験案です。物理物の製作や実機検証を終えていないものは構想段階です。

1. **現場で生成 → NFCで持ち帰る**  
   対象物から作品を生成し、その場でNFCタグへURLを書き込む。参加者はタグを持ち帰り、後日かざして再生する。

2. **現場で生成 → QRカードで持ち帰る**  
   作品URLのQRをカードに印刷または表示する。iPhone・Androidを問わず、標準カメラから作品を開く。

3. **展示物ごとの作品を呼び出す**  
   展示物にNFCまたはQRを添え、来場者が触れた・選んだ物に対応する音と映像を再生する。

4. **画像ターゲットに作品を重ねる**  
   印刷図案へスマートフォンを向けると、その面に生成ビジュアルが重なって見える。見失った場合は通常再生へ戻す。

5. **レーザー彫刻QRから後日再生する**  
   廃材へ作品URLのQRを彫刻し、物そのものを作品の入口にする。読み取り安定性は素材別試作で判断する。

6. **レーザー彫刻AR + NFCの二重導線**  
   彫刻図案でARを見せ、暗所や反射で認識しづらい場合はNFCで通常再生へ切り替える。

7. **素材ごとに音色を変える**  
   アルミ、プラスチック、紙などの認識結果を、音階・速度・色・形の違いへ割り当てる。分類精度は実物データで調整する。

8. **同じ素材から毎回違う作品を作る**  
   素材の規則を保ちながら、撮影ごとの偶然性で異なるSeedを作る。ツアー参加者ごとに個別の作品を残せる。

9. **展示会場で生成 → 自宅で再生する**  
   会場では短時間で生成し、URL・QR・NFCのいずれかを持ち帰る。自宅では同じSeedから作品を再構成する。

10. **作品カードとデジタル再生を併用する**  
    PNGを記念画像として保存し、QRやNFCから動きと音のある完全版を開く。静止画とデジタル体験を一組にする。

## 5. 端末・運用上の注意

- カメラ利用には HTTPS のページ、または開発用の localhost が必要です。
- 音はブラウザの制約上、利用者が開始・再生ボタンを押した後に鳴ります。
- Web画面からNFCへ書き込む運用は主にAndroid Chrome向けです。iPhoneでは、別の端末やアプリで書き込んだタグを読み取る運用を想定します。
- iPhoneとAndroidではAR・NFCの対応範囲が異なるため、QR・URLによる通常再生を共通の予備導線として残すのが安全です。
- AI認識の精度は、対象物だけでなく照明、背景、距離、反射の影響を受けます。展示条件をそろえた学習と実機確認が必要です。
- レーザー彫刻は、素材の色・反射・焦げ・線幅によって読み取りやすさが変わります。現時点では実物試作前の構想です。

## 6. 次の検証ステップ案

1. 更新後UIをiPhone SafariとAndroid Chromeで確認する
2. 実際に使う対象物を3種類程度に絞り、撮影条件と学習データを準備する
3. 分類モデルを配置し、誤認識率と認識にかかる時間を記録する
4. 別端末でSeed URLとQRから同じ作品が再生されるか確認する
5. NFCタグの書き込み・読み取りをAndroidとiPhoneで役割分担して確認する
6. 印刷した画像ターゲットでAR追跡を試し、成立後にレーザー彫刻素材へ進む
7. QR、NFC、ARのどれを主導線にするか、体験価値と運営負荷を比較して決定する
