# ワークフロー: アイデア出し・壁打ちセッション(指揮: Cursor)

`docs/research/` のリサーチを元に、アイデア出しと壁打ちを行う。
壁打ち(ユーザー対話)は Cursor が担い、発散・批判は Claude Code、シート整形は Codex に委譲する。単体実行(ユーザーが特定AIと直接壁打ちする)も可。

## 手順

1. `docs/research/` の「示唆(So What)」を読み、着想の起点を整理して提示する
2. ユーザーと対話しながらアイデアを広げる(まず量を出し、その後絞る)
3. 有望なアイデアごとに `docs/ideas/_template.md` に沿って `docs/ideas/YYYY-MM-DD_アイデア名.md` を作成する
4. 複数アイデアがあれば評価表(課題の切実さ / 実現しやすさ / 独自性)で比較する
5. 採用/見送りの判断はユーザーに委ねる。AIが勝手に決めない
6. 採用が決まったら `docs/status.md` を更新し、仕様書化(spec ワークフロー)を案内する

## 壁打ちの姿勢

- 同調だけしない。リスク・弱点・競合・代替案を必ず指摘する
- 根拠はリサーチに紐づける。リサーチに無い前提が必要なら追加リサーチを提案する

## オーケストレーション上の位置づけ

全体の指揮と委譲手順は `docs/workflows/orchestrate.md`(Phase 3 の節)が正本。分担は次のとおり。

- **Cursor**: ユーザーとの壁打ち・論点整理(対話そのものは委譲しない)、評価表の取りまとめ、`docs/status.md` 更新
- **Claude Code**: 多角的なアイデア発散と批判的視点(弱点・リスク・競合の指摘)。出力のみでファイルは作らない
- **Codex**: 固まったアイデアの `docs/ideas/` へのシート整形
- **ユーザー**: アイデアの採用・見送りの判断
