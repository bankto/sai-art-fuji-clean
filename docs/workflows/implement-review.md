# ワークフロー: 実装レビュー(担当: Claude Code)

実装されたコードを仕様書と突き合わせてレビューし、レビューレポートを作成する。

## 手順

1. 対象の仕様書(`docs/specs/`)と実装(`apps/対象/`)を読む
2. 次の観点でレビューする
   - 仕様との整合: 仕様書の機能・完了条件を満たしているか。仕様に無い機能が勝手に足されていないか
   - バグ: ロジックの誤り、エッジケースの考慮漏れ、エラーハンドリング不足
   - セキュリティ: シークレットのハードコード、XSS などの基本的な脆弱性
   - アクセシビリティ: セマンティクス、キーボード操作、aria 属性
   - 可読性・保守性: 過剰な複雑さ、重複、命名
3. 可能であれば lint・ビルドを実行して結果を確認する(コードの修正はしない)
4. `docs/reviews/_template.md` に沿って `docs/reviews/YYYY-MM-DD_対象名_review.md` を作成する
5. 判定(承認 / 条件付き承認 / 要修正)を明記する

## 注意

- コードは直接修正しない。指摘と修正案はレポートに書く(反映は Codex の役割)
- 指摘には該当ファイル・該当箇所を明記し、根拠と修正案を添える
- 動かして確認できたこと・できなかったこと(未検証)を区別して書く
